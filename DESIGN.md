# 《焚天问道》— 沉浸式文字修仙模拟游戏 设计文档

> working title，可改。一款以"天才陨落—逆境重修—问鼎苍穹"为主线的分支叙事修仙手游。
> 玩法核心：在关键人生节点做选择，进入不同人生路径（正道 / 魔道 / 逍遥）。

---

## 1. 立项要点

| 项 | 决策 | 理由 |
|---|---|---|
| IP | **原创世界观，斗破"同味不同名"** | App Store 审核会拒绝直接侵权的角色名/专有名词/剧情。我们借用"套路"，不借用"资产"。 |
| 第一阶段形态 | **Web 文字原型**（本仓库 `prototype/`） | 剧情=数据，引擎=代码，两者分离。Windows 上双击即玩，改剧情零成本，剧情数据可迁移到任意最终引擎。 |
| 最终上架技术栈 | **Expo (React Native) + EAS Build** | 你在 Windows 且无 Mac。EAS 云构建可在**没有 Mac 的情况下打 iOS 包并提交 App Store**。剧情数据(JSON)直接复用本原型。 |
| 备选技术栈 | Flutter / Unity+Ink / 原生 SwiftUI | Flutter 需 Mac 出包；Unity 适合后期加战斗/立绘；SwiftUI 体验最好但强制 Mac。 |

## 2. 世界观（原创，致敬斗破）

- **力量体系**：修士引"焚天玄气"淬体凝魂，分九大境界，每境九重。
- **境界**：凝气 → 筑基 → 金丹 → 元婴 → 化神 → 炼虚 → 合体 → 大乘 → 渡劫（→ 传说中的"仙人"）。
- **异火榜**："天焚录"记载天地间二十三种异火，可吞噬炼化、增幅战力（对应斗破异火榜）。
- **炼药师**：以玄气为引、异火为炉，炼制丹药；炼药师地位尊崇（对应斗破炼药师体系）。
- **戒中老者**：主角祖传戒指中封印着一缕残魂——上古炼药宗师 **墨渊**（对应"药老"），三年前因机缘暗附主角，意外压制了他的修为。

### 主线钩子
主角 **林夜**（玩家可改名），昔日 **林家** 百年一遇的天才（凝气九重），三年前修为离奇倒退至凝气三重，沦为废物、备受嘲讽。云家小姐 **云清霜** 上门退婚——这是第一个关键抉择节点。当夜戒指中的墨渊残魂苏醒，主角踏上逆袭之路。

## 3. 玩法系统

### 3.1 属性
- **根骨**：影响修炼速度与体魄上限
- **悟性**：影响功法/炼药领悟、战斗触发"顿悟"
- **神识**：影响异火驾驭、炼药成功率、感知危险
- **气运**：影响奇遇、暴击、随机事件好坏

### 3.2 道心 / 心境（分支轴）
每个抉择累积 **道心值**（-100 魔 ↔ +100 正）。临界点解锁不同人生路径：
- **正道仙途**：守护苍生、宗门长老线
- **魔道焚天**：弑敌夺宝、魔domain主线
- **逍遥隐世**：避世问道、双修/经商/炼药富甲线

### 3.3 资源
玄气（修为经验）、灵石（货币）、丹药、功法、异火、人际关系（好感度/势力声望）。

### 3.4 抉择节点设计原则
- 每个大节点至少 1 个"无对错、只关乎人生走向"的选择
- 选择应触发**可见的长期后果**（标记 flag，后续剧情回调）
- 数值门槛选项（需根骨≥X / 持有某物）制造 build 差异

## 5. 商业化（合规、克制）

- **基础免费 + 内购**：解锁后续章节卷、加速修炼、外观/书房皮肤
- **激励广告**（可选）：领悟丹、洗点机会
- 合规红线：不做开箱/赌博机制误导未成年；明确年龄分级；隐私合规（不采集敏感数据）。

---

## 6. App Store 上架清单（Windows → iOS，无需 Mac）

1. 注册 **Apple Developer Program**（$99/年）
2. 用 **Expo** 重写前端（剧情 JSON 直接复用 `prototype/js/story.js` 的数据结构）
3. `eas build -p ios` 云端出包；`eas submit` 提交
4. App Store Connect 填写：分级(12+ 含轻度暴力/魔幻)、隐私清单、截图、描述
5. 审核注意：原创 IP、无侵权素材、内购合规、有"恢复购买"按钮

## 7. 仓库结构

```
prototype/        可直接游玩的 Web 文字原型
  index.html
  css/style.css
  js/engine.js    存档/渲染/数值/分支引擎（内容无关）
  js/story.js     剧情内容（数据，最终可迁移）
DESIGN.md         本文件
```

运行：直接用浏览器打开 `prototype/index.html` 即可。
